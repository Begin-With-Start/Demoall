多线程的基本用法，线程同步相关例子；

    多线程出现在安卓中，最多的来源于文件操作，数据库，aidl，网络，大文件处理，大图片处理，编码解码，安全验证，
    等等耗时操作。进行耗时操作的时候不能够让ui进程等待操作完成再给用户反馈，而是应该ui线程之外进行处理，然后再ui线程刷新页面给到用户最终的结果反馈；
    那么在这其中就涉及到两个知识点，一个是子线程的新建与管理，另一个就是线程间的通讯问题也就是同步问题；之后的一些讨论点和源代码基本都是围绕着解决这两个问题来展开的；

    而Android 程序的大多数代码操作都必须执行在主线程，例如系统事件(例如设备屏幕发生旋转)，输入事件(例如用户点击滑动等)，程序回调服务，UI 绘制以及闹钟事件等等。那么我们在上述事件或者方法中插入的代码也将执行在主线程。
